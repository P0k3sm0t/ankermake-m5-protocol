# AnkerMake M5 Protocol (ankerctl)

## Project Overview

`ankerctl` is a Command-Line Interface (CLI) and Web UI tool designed to monitor, control, and interface with AnkerMake M5 3D printers. It provides an open-source alternative to the proprietary AnkerMake software, allowing for local LAN control, integration with slicers (PrusaSlicer, OrcaSlicer), and raw protocol access.

The project is built primarily in Python and utilizes a custom library named `libflagship` for handling the various communication protocols (MQTT, PPPP, HTTP) used by the printers.

## Architecture & Key Components

*   **`ankerctl.py`**: The main entry point for the CLI application. It uses the `click` library for command handling.
*   **`libflagship/`**: The core library containing the implementation of communication protocols.
    *   `mqtt.py`, `pppp.py`, `httpapi.py`: Protocol implementations.
    *   `amtypes.py`: AnkerMake specific types and data structures.
*   **`web/`**: The Flask-based web application providing a UI for the printer.
    *   Uses `flask-sock` for WebSocket support (real-time status, video streaming).
    *   `service/`: Backend services for the web app.
*   **`cli/`**: Contains the command logic for the `ankerctl` CLI.
*   **`templates/`**: Source templates used by `transwarp` to generate code in `libflagship` and `static`.

## Development Setup

### Prerequisites

*   Python 3.10+
*   `pip`
*   `transwarp` (installed via `make install-tools`)

### Installation & Dependencies

1.  **Install Python Dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

2.  **Install Development Tools:**
    ```bash
    make install-tools
    ```

### Building & Code Generation

This project uses a tool called `transwarp` to generate some source files from templates.

*   **Update/Generate Code:**
    ```bash
    make update
    ```
    This command updates `libflagship` and `static` files based on templates in `templates/`.

*   **Check Diff (Dry Run):**
    ```bash
    make diff
    ```

*   **Clean Artifacts:**
    ```bash
    make clean
    ```

## Running the Application

### CLI Usage

The CLI is run via `ankerctl.py`.

*   **Help:**
    ```bash
    ./ankerctl.py --help
    ```
*   **Import Configuration:**
    ```bash
    ./ankerctl.py config import <path_to_login.json>
    ```
*   **Start Web Server:**
    ```bash
    ./ankerctl.py webserver run
    ```
*   **Search for Printers (LAN):**
    ```bash
    ./ankerctl.py pppp lan-search
    ```

### Web Interface

*   **Run via CLI:** ` ./ankerctl.py webserver run`
*   **Access:** Open `http://localhost:4470` in your browser.
*   **Docker:** `docker-compose up` (configured via `docker-compose.yaml`).

## Configuration

*   **Storage:** Configuration is stored in a `default.json` file managed by `ankerctl`.
*   **Import:** Configuration is typically imported from the official AnkerMake Slicer's `login.json` or `user_info` file.
*   **CLI Config:** `cli/config.py` handles loading and saving configuration.

## Key Files & Directories

*   `ankerctl.py`: Main executable.
*   `requirements.txt`: Python dependencies.
*   `Makefile`: Build automation.
*   `libflagship/`: Protocol library source.
*   `cli/`: CLI command implementations.
*   `web/`: Web application source.
*   `documentation/`: Project documentation.
*   `examples/`: Example scripts for using the library.
*   `templates/`: Code generation templates.

## Conventions

*   **Code Style:** Follows standard Python PEP 8 conventions.
*   **Logging:** Uses `cli.logfmt` for formatted logging.
*   **Safety:** `ankerctl.py` includes an `--insecure` flag to disable TLS verification (useful for debugging but discouraged in production).
*   **Generators:** Do not manually edit files that are generated by `transwarp` (check `Makefile` to see which targets are generated). Edit the `templates/` instead.
